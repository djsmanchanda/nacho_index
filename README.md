# Nacho Index

A highly stylized personal leaderboard for reviewing nachos, chips, and snacks — with weighted scoring, analytics, auto-ranked leaderboards, and a snack-lab aesthetic.

Built for **Cloudflare Pages + D1** using **Next.js App Router**, **Drizzle ORM**, **Tailwind CSS**, **shadcn-style UI**, **Framer Motion**, and **Recharts**.

## Features

- **Leaderboard** — animated cards, top-3 podium, sort/filter by score, brand, date
- **Add review** — sliders for 7 rating dimensions, live weighted score, auto product image fetch + manual override
- **Review detail** — hero image, radar chart, breakdown cards, AI-style verdict narrative, derived stats
- **Analytics** — brand averages, score distribution, category champions
- **Scoring** — weighted formula with crunch efficiency, grease penalty, rebuy index
- **Verdicts** — autogenerated tags like *Scientifically crunchy*, *Salt attack*, *Elite gas station engineering*

## Weighted score formula

```
weighted =
  overall × 0.25 +
  taste × 0.35 +
  crunch × 0.15 +
  aftertaste × 0.10 +
  rebuy_value × 0.10 +
  salt_balance × 0.05
```

## Quick start (local)

```bash
npm install

# Apply D1 migrations to local SQLite
npm run db:migrate:local

# Optional: seed sample reviews
npm run db:seed

# Dev server (D1 binding via OpenNext dev integration)
npm run dev
```

Open [http://localhost:3000](http://localhost:3000).

> **Note:** Server actions require the D1 `DB` binding. `next dev` uses `@opennextjs/cloudflare` dev integration to attach a local D1 database. If you see binding errors, run `npm run cf:preview` instead.

## Environment variables

Copy `.env.example` to `.env.local` for remote migrations only:

```bash
cp .env.example .env.local
```

| Variable | Purpose |
|----------|---------|
| `CLOUDFLARE_ACCOUNT_ID` | Drizzle Kit remote migrations |
| `CLOUDFLARE_D1_DATABASE_ID` | Your D1 database UUID |
| `CLOUDFLARE_API_TOKEN` | API token with D1 edit permissions |

Update `wrangler.jsonc` → `d1_databases[0].database_id` after creating the database.

## Database commands

```bash
# Generate new migration from schema changes
npm run db:generate

# Apply migrations locally
npm run db:migrate:local

# Apply migrations to production D1
npm run db:migrate:remote

# Seed sample data (local)
npm run db:seed
```

### Create production D1

```bash
npx wrangler d1 create nacho-index-db
```

Copy the returned `database_id` into `wrangler.jsonc`, then:

```bash
npm run db:migrate:remote
```

## Cloudflare Pages deployment

### 1. Build & preview locally

```bash
npm run cf:preview
```

### 2. Deploy to Cloudflare

```bash
npm run cf:deploy
```

Or connect the repo in **Cloudflare Dashboard → Workers & Pages → Create → Pages** and set:

| Setting | Value |
|---------|--------|
| Build command | `npm run cf:build` |
| Build output | `.open-next` (handled by adapter) |
| Node version | 20 |

Bind **D1** in the project settings: variable name `DB`, database `nacho-index-db`.

### 3. Run production migrations

```bash
npm run db:migrate:remote
```

## Project structure

```
src/
  app/                 # Routes (leaderboard, add, analytics, review/[id])
  app/actions/         # Server actions (reviews CRUD, previews)
  components/          # UI, leaderboard, charts, analytics
  lib/
    db/                # Drizzle schema + getDb()
    scoring.ts         # Weighted formula + derived stats
    verdicts.ts        # Autogenerated verdicts & narratives
    image-fetch.ts     # Product image search + cache
    analytics.ts       # Dashboard aggregations
drizzle/migrations/    # SQL migrations
wrangler.jsonc         # D1 binding + Worker config
open-next.config.ts    # OpenNext Cloudflare config
```

## Image fetching

Product images are resolved via:

1. **Local cache** (`image_cache` table)
2. **Open Food Facts** API search
3. **Wikimedia Commons** thumbnail API
4. **Placeholder** SVG fallback

Manual URL override is supported on the add-review form.
